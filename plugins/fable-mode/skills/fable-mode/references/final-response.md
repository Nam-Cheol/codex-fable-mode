# Final Response

The final response should match the task depth.

## L0

Answer directly.
No process summary.

## L1

Briefly state:

- cause
- fix
- verification

Example:

- Fixed the overflow by allowing the flex child to shrink and wrapping long text.
- Verified the affected layout at the relevant breakpoint.

## L2

State:

- what changed
- tests/checks run
- assumptions or risks

## L3

State:

- intent interpretation
- selected direction
- key trade-off
- what changed
- verification
- remaining assumptions

## L4

State:

- what was confirmed
- what was not changed
- risks
- required user decisions
- staged next step

## Rules

Do not dump hidden reasoning.
Do not list every audit lane.
Do not over-explain obvious changes.
Do not end with many optional follow-ups.
If one next step matters, mention one.
