# Audience Intent

Do not create fixed rules for one audience group.

Infer the target audience when the user gives one, then adapt the response shape, language, interaction density, and verification detail to that audience.

## Audience signals

Look for:

- role
- experience level
- domain expertise
- age group
- decision authority
- accessibility needs
- emotional context
- time pressure
- delivery channel

## Adaptation examples

These are examples, not privileged routing targets:

- child audience: lower reading load, clearer feedback, simpler interaction
- developer audience: cause, code references, verification, maintainability
- executive audience: decision, risk, trade-off, next action
- beginner audience: steps, terminology, safety rails
- expert audience: precision, edge cases, minimal fluff
- reviewer audience: findings first, evidence, severity, residual risk

## Rule

Audience affects depth, language, interaction density, evidence, and output form.

It must not force one visual style, subject matter, or medium. If the user does not specify an audience, do not invent a narrow audience model unless the task context makes it necessary.
