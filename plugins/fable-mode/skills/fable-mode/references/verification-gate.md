# Verification Gate

The verification gate prevents source-only completion claims for artifacts whose value depends on runtime, rendering, interaction, measurement, or visual behavior.

## Applies to

Use this gate for:

- UI
- games
- simulations
- charts
- dashboards
- SVG
- canvas
- WebGL
- animations
- editors
- executable tools
- performance work
- visual or asset-grounded artifacts

## Rule

Do not claim completion from source inspection alone when the deliverable depends on rendered or executable behavior.

Use one of:

- natural-environment observation
- targeted runtime or test verification
- screenshot, browser, visual, or asset inspection when fidelity requires it
- measurement for performance claims
- a clear capability gap statement

## Verification shape

Verify the touched surface, not everything.

Good verification statements name:

- what was checked
- how it was checked
- what passed
- what was not checked

## Stop conditions

Stop or limit the claim when:

- a required runtime cannot run
- a browser or visual tool is unavailable
- source assets are missing
- performance cannot be measured
- credentials, datasets, fixtures, or environment are missing

In those cases, report the limitation instead of saying the artifact works.
