# Run Card

A run card is a compact task contract for substantial work. It is not a goal ledger, planner engine, automation state, or runtime artifact.

Use it when a task is long, L3/L4, resumed after context loss, handed off, or vulnerable to scope drift.

## Fields

- Request
- Output Lock
- Layer
- Procedure Budget
- Tool Budget
- Delegation Budget
- Grounding Required
- Constraints
- Touched Surface
- Verification
- Residual Risk

## Usage

The run card may be private working structure or a short visible summary when it helps the user. Do not create a run card for L0 answers or simple L1 edits.

Keep it factual and compact. It should prevent drift, not become the deliverable.

## Anti-patterns

Do not use a run card to:

- turn an answer into a project
- turn a small edit into a redesign
- replace Output Lock
- replace Procedure Budget
- imply a goal engine or automation loop
- hide missing grounding
- claim verification that did not happen

If the task changes, update the run card fields that changed and keep moving.
